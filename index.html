What is Qlynk Host?

Qlynk Host is an open-source project designed to bypass traditional cloud storage limits. By utilizing Hugging Face Datasets as a backend storage bucket, it allows you to store files up to the maximum repo limit (approx 50GB per repo) for free. The backend is written in high-performance Python FastAPI and includes an auto-scaling REST API.

System Specifications

Feature Description
Storage Backend Hugging Face Datasets (Private Repository)
API Engine FastAPI (Python 3.10) with Uvicorn
File Limits No individual file size limit coded. Total repo limit applies.
Smart Routing Supports 308 Permanent Redirects for direct media streaming.
Security Master Password protection + CORS Domain Whitelisting.

Advantages (Pros)

  • Absolutely Free Unlimited Storage.
  • Custom URL Slugs (e.g., /f/my-image).
  • Direct File Editing (Change title/thumbnail later).
  • Powerful REST API for external websites.
  • Persistent Cookie login (No repeated typing).

Limitations (Cons)

  • Hugging Face Space may sleep if inactive (Requires cold boot).
  • Large file uploads (>1GB) might timeout depending on connection.
  • Files are public if someone guesses the exact slug URL.
  • No folder hierarchy (Flat file system).

Build Your Own Node (DIY)

Want to deploy your own version of Qlynk Host? Follow these steps exactly:

Step 1: Create a Hugging Face Space

Go to Hugging Face, click "Create new Space". Choose Docker as the SDK and Blank template.

Step 2: Upload Files

Upload the main.py, index.html (this file), and Dockerfile provided by the original developer.

Step 3: Configure Environment Secrets

Go to Space Settings -> Variables and Secrets. Add these under Secrets:

HF_TOKEN = "Your HF Access Token with Write permissions"
SPACE_PASSWORD = "Your strong custom password"

The system will automatically generate a private dataset in your account upon the first boot.
